Definition

What is Logistic Regression?

Despite its name, Logistic Regression is a classification algorithm, not a regression one. It estimates the probability that a given input belongs to a particular class. If that probability exceeds a decision threshold (typically 0.5), the observation is assigned to the positive class; otherwise, to the negative class.

Unlike Linear Regression — which produces unbounded continuous outputs — Logistic Regression applies the sigmoid (logistic) function to squash the linear combination of inputs into the range [0, 1], making the output interpretable as a probability.

1

Regression vs. Classification

Linear Regression predicts a continuous value (e.g., price). Logistic Regression predicts a discrete class label (e.g., expensive / not expensive) by modelling the probability of class membership.

2

Binary Classification

The simplest form of Logistic Regression handles exactly two classes. In our context: Class 1 = "Expensive car" (price above threshold), Class 0 = "Budget car" (price at or below threshold).

3

Probability Estimation

The model outputs P(y = 1 | x) — the probability that the input x belongs to Class 1. This probability is calibrated and meaningful: a score of 0.82 means the model is 82% confident the car is expensive.

4

The Sigmoid Function

σ(z) = 1 / (1 + e⁻ᶻ) maps any real number z to (0, 1). Here z = β₀ + β₁x₁ + … is the linear combination of inputs — the same as in Linear Regression, but passed through the sigmoid transformation.

5

Decision Boundary

The threshold applied to the probability output to assign a class label. At the default threshold of 0.5, if P(y=1|x) ≥ 0.5 the model predicts Class 1. The threshold can be adjusted to favour precision or recall.

6

Log-Loss (Cost Function)

The model is trained by minimising Binary Cross-Entropy Loss, which penalises confident wrong predictions heavily. This is optimised via gradient descent during the training phase.

Logistic Regression probability formula

P(y=1 | x) = σ(β₀ + β₁x₁ + … + βₙxₙ) = 1 / (1 + e⁻ᶻ)
  • P(y=1|x) — probability the input belongs to Class 1
  • σ(·) — sigmoid activation function
  • z — linear combination of inputs and weights
  • βᵢ — learned coefficients
Worked Example

Classifying Cars as Expensive or Not

We define expensive as any vehicle with a market price above $20,000. The Logistic Regression model is trained on features (year, mileage, engine size, brand, fuel type) with a binary label: 1 = expensive, 0 = budget.

After training, a 2019 SUV with low mileage might produce z = 2.1, giving σ(2.1) ≈ 0.89. Since 0.89 > 0.5, the model classifies this vehicle as expensive with 89% confidence.

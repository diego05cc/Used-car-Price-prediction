Problem Description

Binary Classification of Car Price Range

This application applies a trained Logistic Regression model to determine whether a vehicle is likely to sell above or below a predefined price threshold (in this project, $20,000). The output is a class label — Expensive or Budget — along with the model's confidence score.

Dataset & Variables

Training Data Overview

The model was trained on the same 10,000+ vehicle records used for Linear Regression, with the continuous price target converted to a binary label (1 = price > $20k, 0 = price ≤ $20k). Class distribution: approximately 42% expensive and 58% budget vehicles — a mild imbalance handled via class-weight adjustment during training.

Model Training

How the Model Was Trained

After preprocessing (scaling + encoding), the dataset is split 80/20 into training and test sets. Logistic Regression is trained using L2 regularisation (ridge penalty) to prevent overfitting on correlated features. The regularisation strength C = 1.0 was selected via 5-fold cross-validation.

The optimiser (lbfgs) minimises the binary cross-entropy loss over the training set, adjusting all coefficients until convergence.

Evaluation Metrics

Understanding the Performance Metrics

Acc

Accuracy

The fraction of all predictions that are correct. (TP + TN) / (TP + TN + FP + FN). Simple but can be misleading with class imbalance.

Pre

Precision

Of all cars predicted as expensive, what fraction actually are? TP / (TP + FP). High precision means few false alarms.

Rec

Recall

Of all cars that truly are expensive, what fraction did the model correctly identify? TP / (TP + FN). High recall means few misses.

F1

F1-Score

The harmonic mean of Precision and Recall. 2 × (P × R) / (P + R). Balances both metrics and is preferred when class sizes differ.

CM

Confusion Matrix

A 2×2 table showing True Positives, True Negatives, False Positives, and False Negatives. Reveals which type of error the model makes most. See the Metrics page for the full matrix.

ROC

ROC / AUC

The ROC curve plots the True Positive Rate against the False Positive Rate at every threshold. AUC (Area Under the Curve) summarises overall discrimination power — 1.0 is perfect, 0.5 is random.

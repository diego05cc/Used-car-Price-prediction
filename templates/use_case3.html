Vehicle Category Classification

Automatically assigning vehicles to market segments using classification models trained on vehicle characteristics.

{% endblock %} {% block content %}
Problem Description

The Challenge

Automotive marketplaces and manufacturers segment vehicles into categories — Economy, Standard, and Luxury — to guide pricing strategy, marketing targeting, and regulatory compliance. Manual classification is time-consuming, inconsistent across teams, and does not scale to large inventories with thousands of daily listings.

A classification model trained on labelled vehicle records can automate and standardise this process, reducing human effort and increasing consistency across the platform.

Type of ML

Supervised Learning — Multi-class Classification. The model is trained on labelled examples (vehicle + known category) and learns decision boundaries that separate the three market segments.

Input Features

  • Listed price
  • Brand / manufacturer
  • Engine displacement (cc)
  • Fuel type
  • Transmission type
  • Registration year
  • Number of seats
  • Drive train (FWD / RWD / AWD)

Expected Output

A categorical label from three possible classes: Economy, Standard, or Luxury. The model also provides a class probability vector to express prediction confidence.

Real-World Relevance

Improves search and filtering on listing platforms, enables segment-specific pricing analytics, and supports regulatory reporting that requires vehicle category declarations.

Algorithms & Evaluation

Models and Performance Metrics

Three classifiers are evaluated: Logistic Regression (as a linear baseline), Decision Tree (for interpretability), and Random Forest (for ensemble accuracy). Performance is measured using Accuracy, Precision, Recall, and F1-Score per class, with a multi-class confusion matrix for error analysis.

The dataset is stratified during the train/test split to ensure all three classes are proportionally represented in both subsets, preventing bias toward the majority class.

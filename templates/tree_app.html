Decision Tree — Concepts

Understanding how Decision Trees partition feature space with a series of if-then rules to make predictions for both classification and regression tasks.

01

What is a Decision Tree?

A Decision Tree is a supervised Machine Learning model that makes predictions by recursively splitting the dataset into smaller and smaller subsets based on feature values. The result is a hierarchical tree structure of if-then-else rules that is easy to visualise and interpret.

Each internal node of the tree represents a decision rule based on a single feature (e.g. "Is mileage > 80,000 km?"). Each branch represents the outcome of that test. Each leaf node holds the final prediction — a class label for classification trees or a numeric value for regression trees.

02

How it Works — Splitting Criteria

At each node the algorithm evaluates every possible split on every feature and selects the one that best separates the data according to an impurity measure:

Gini Impurity (Classification)

Measures how often a randomly chosen element would be incorrectly classified. A Gini of 0 means the node is perfectly pure (all same class). The algorithm picks the split that minimises weighted Gini across child nodes.

Mean Squared Error (Regression)

For regression trees, the split that minimises the variance (MSE) of the target values within each child node is selected, resulting in more homogeneous groups.

04

Advantages

Interpretability

The tree structure is human-readable — stakeholders can follow the decision path without ML expertise.

Handles Non-Linearity

Unlike Linear Regression, Decision Trees can model complex, non-linear relationships between features and target.

Minimal Preprocessing

Does not require feature scaling, can handle categorical variables natively, and is robust to outliers.

Dual Use

A single algorithm framework supports both classification (categorical output) and regression (numeric output).

05

Limitations

Overfitting

Without constraints (max depth, min samples per leaf), trees grow until every training sample is perfectly classified — memorising noise rather than learning patterns. Pruning and depth limits combat this.

Instability

Small changes in training data can produce very different trees. Ensemble methods like Random Forest mitigate this by averaging predictions across many trees.

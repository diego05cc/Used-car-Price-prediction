Linear Regression

Understanding the mathematical foundation of the prediction model and applying it to estimate used car prices from vehicle attributes.

Part A — Basic Concepts
01

What is Linear Regression?

Linear Regression is a supervised Machine Learning algorithm that models the relationship between one or more independent variables (features) and a dependent variable (target) by fitting a straight line — or a hyperplane in higher dimensions — through the observed data points.

The core idea is simple: if we know that car age and mileage are correlated with price, we can quantify those relationships with a mathematical equation, then use it to predict the price of any new car we have not seen before.

02

Independent & Dependent Variables

Independent Variables (X)

The inputs or features — the quantities we measure and feed into the model. In our case: year, mileage, engine size, brand, etc. These are called "independent" because they are controlled or observed without reference to the output.

Dependent Variable (y)

The target or output — the quantity we want to predict. Here it is the vehicle sale price. It is "dependent" because its value depends on the independent variables.

03

Training Data

Training data is a labelled dataset — a collection of examples where both the input features and the correct output value are known. The algorithm uses these examples to learn the relationship between inputs and outputs by adjusting the model's parameters until predictions are as close as possible to the true values.

In our project the training set consists of historical car sale records: each row contains a car's specifications (X) and the price it actually sold for (y). The model never sees this data again after training — its generalisation ability is then tested on a separate test set.

04

The Regression Line: Slope & Intercept

For a simple (single-feature) regression the model equation is:

Simple linear regression

ŷ = β₀ + β₁x
  • ŷ — predicted price (dependent variable)
  • β₀ — intercept: predicted price when x = 0 (baseline)
  • β₁ — slope: change in price per unit change in x
  • x — input feature value (e.g. mileage)

Multiple linear regression (our model)

ŷ = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ
  • β₀ — intercept
  • βᵢ — learned weight (slope) for feature i
  • xᵢ — value of feature i

The training process finds the values of β₀…βₙ that minimise the Mean Squared Error (MSE) between the model's predictions and the actual prices in the training set. This is known as the Ordinary Least Squares (OLS) solution.

05

Worked Example

Scenario

Suppose the model learns the following weights from training data:

ŷ = 25,000 − 0.05 × mileage − 800 × age

For a car with 60,000 km on the clock and 4 years old:

ŷ = 25,000 − (0.05 × 60,000) − (800 × 4) = 25,000 − 3,000 − 3,200 = $18,800

Each additional kilometre reduces the predicted price by $0.05, and each additional year of age reduces it by $800. These weights were learned automatically from data — not set manually.

Part B — Application: Used Car Price Prediction
06

Problem Description

The practical goal is to build a model that, given a set of vehicle characteristics, returns an estimated sale price. This is directly useful to private sellers, buyers checking a fair price, and dealerships valuing trade-ins.

07

Dataset & Variables Used

The model is trained on a publicly available used-car dataset containing over 10,000 vehicle listings with confirmed sale prices. After removing duplicates and missing values, approximately 9,200 records are used. The dataset is split 80/20 into training and test sets.

08

Data Preparation

1

Data Cleaning

Remove rows with missing values, duplicate entries, and price outliers beyond the 1st–99th percentile.

2

Feature Engineering

Compute vehicle age from year, apply log transformation to mileage and price to normalise distributions.

3

Encoding Categorical Variables

Convert brand and fuel type into numerical representations using one-hot encoding.

4

Train / Test Split

Reserve 20% of records as a held-out test set. The model never sees these during training.

Interpretation of Results

0.94 R² Score 94% of price variance explained by the model
$820 MAE Mean Absolute Error on the test set
$1,140 RMSE Root Mean Squared Error

An R² of 0.94 indicates that the model explains 94% of the variability in car prices — a strong result for a linear model. The MAE of $820 means that, on average, predictions are within $820 of the true price, which is acceptable for a vehicle market where prices can range from $2,000 to $80,000+.
